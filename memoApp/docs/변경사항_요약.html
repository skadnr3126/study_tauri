FLOW MEMO · 1차 구현 보고서

Flow 기본 구조를 만들었습니다.

파일 저장과 Canvas는 아직 넣지 않고, Flow 안에서 생각(Block)을 이어 쓰고 Branch로 분기·이동·삭제하는 기본 조작을 구현했습니다.

현재 상태: 모든 데이터는 메모리에만 있습니다. 앱을 종료하면 사라지며, Markdown·JSON 저장은 2차 목표입니다.

구현된 기능

Flow 관리

새 Flow 생성, 목록 전환, Flow 이름 변경을 지원합니다.

Block 편집

선택적 제목과 Markdown 본문을 편집합니다. 제목이 없으면 본문 첫 줄을 표시 제목으로 사용합니다.

Tree 조작

다음 Block 추가, Branch 생성, Block 순서 변경, 다른 Branch 이동, Branch 이동을 지원합니다.

안전 장치

하위 Branch로의 순환 이동을 막고, Block 삭제 시 이어지는 하위 흐름을 함께 제거합니다.

핵심 데이터 구조

WorkspaceState
├─ blocks: Map<BlockId, Block>
└─ flows: Map<FlowId, Flow>

Flow
├─ rootBranchId
└─ branches: Map<BranchId, Branch>

Branch
├─ itemIds: BlockId[]
└─ childBranchIdsByBlockId: Map<BlockId, BranchId[]>

별도 Node 모델은 없습니다. Block ID가 Flow tree의 유일한 항목이며, Branch는 순서와 분기를 표현하는 컨테이너입니다.
