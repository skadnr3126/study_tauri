# 11. 사용자 구조와 시스템 구조

현재 1차 구현에서 사용자가 직접 다루는 구조는 `Block`, `Flow`, `Branch`까지다. Link와 Inbox는 Flow tree 조작이 검증된 뒤 추가한다.

사용자에게 구조를 강제해서는 안 된다.

사용자가 기본적으로 다루는 것은 다음 정도면 충분하다.

```text
Block
Flow
Branch
Link
Inbox
```

앱은 내부적으로 다음을 자동 관리할 수 있다.

* 생성 및 수정 시각
* 이동과 삭제 기록
* Undo/Redo
* 파일 경로 매핑
* Backlink
* 최근 접근 기록
* 검색 인덱스
* 로그 Cursor
* 파생 연결
* Export 상태

History, Evolution, Reflection, Reason for Change 같은 개념을 별도 작성 양식으로 강제하지 않는다.

사용자가 본문 안에 자유롭게 쓸 수 있고, 앱은 필요할 때 기존 기록과 조작 내역을 바탕으로 이를 보여줄 수 있다.

Resurfacing 역시 사용자가 작성하는 데이터가 아니라 앱이 과거 기록을 적절한 시점에 다시 보여주는 기능이다.

AI가 추론한 연결이나 변화는 원본이 아니라 다시 계산할 수 있는 파생 정보로 취급한다.
